## 🔄 Workflows

### **Local Development (Current)**

**Docker Compose (What You Use Now):**
```bash
cd Infrastructure/
docker-compose -f docker-compose.complete-system.yml up
```

**After Akashic:**
```bash
# Option 1: Docker Compose (optimized)
cd .akashic/deploy/local/docker/
docker-compose up

# Option 2: Podman (rootless, more secure)
cd .akashic/deploy/local/podman/
podman-compose up

# Option 3: Tilt (fast iteration)
cd .akashic/deploy/local/tilt/
tilt up

# Option 4: Hybrid (recommended)
cd .akashic/deploy/local/scripts/
./start-all.sh
```

### **Cloud Deployment (Terraspace + Juju)**

**Step 1: Provision with Terraspace**
```bash
cd .akashic/deploy/cloud/terraspace/

# Deploy dev environment
terraspace up microk8s -y --var-file=tfvars/dev.tfvars

# Deploy qa environment
terraspace up microk8s -y --var-file=tfvars/qa.tfvars

# Deploy prod environment
terraspace up microk8s -y --var-file=tfvars/prod.tfvars
```

**Step 2: Deploy with Juju**
```bash
cd .akashic/deploy/cloud/juju/

# Deploy to dev
juju deploy bundles/dev-bundle.yml

# Deploy to qa
juju deploy bundles/qa-bundle.yml

# Deploy to prod
juju deploy bundles/prod-bundle.yml
```

**Step 3: Monitor**
```bash
# Check status
kubectl get nodes
juju status

# View monitoring
open http://grafana.dev.yourcompany.com
open http://grafana.qa.yourcompany.com
open http://grafana.prod.yourcompany.com
```

## 💡 Why Each Option?

### **Docker Compose**
- ✅ Simple, what you use now
- ✅ Production-like
- ✅ Works everywhere
- ❌ Slower rebuilds
- ❌ Requires root (less secure)

### **Podman**
- ✅ Rootless (better security)
- ✅ Daemonless (no background service)
- ✅ Drop-in Docker replacement
- ✅ Pod support (Kubernetes-like)
- ✅ Better for CI/CD
- ❌ Slightly different from Docker

### **Tilt**
- ✅ Fast iteration (< 5 second rebuilds)
- ✅ Live reload on code changes
- ✅ Visual dashboard
- ✅ Kubernetes-native
- ❌ Requires more resources
- ❌ Learning curve

### **Hybrid (Recommended)**
- ✅ Best of all worlds
- ✅ Heavy services in Docker/Podman
- ✅ Your services in Tilt (fast iteration)
- ✅ Production-like + Fast development
- ✅ Flexible (switch runtimes easily)

### **Terraspace**
- ✅ One codebase, multiple environments
- ✅ DRY (Don't Repeat Yourself)
- ✅ Easy to maintain
- ✅ Works with Terraform ecosystem
- ✅ Perfect for dev/qa/prod

### **Juju**
- ✅ Charm-based deployment
- ✅ Easy to manage
- ✅ Works with MicroK8s
- ✅ Reusable charms
- ✅ Great for multi-environment

## 🎯 Decision Guide

**Choose Docker Compose if:**
- You want simple setup
- You're already using it
- You don't need fast iteration

**Choose Podman if:**
- You want better security (rootless)
- You don't want a daemon
- You want Kubernetes-like pods

**Choose Tilt if:**
- You want fast iteration
- You're building microservices
- You have a powerful computer

**Choose Hybrid if:**
- You want best of all worlds
- You have heavy services + your services
- You want flexibility

**Choose Terraspace + Juju if:**
- You're deploying to cloud
- You need dev/qa/prod environments
- You want easy charm deployment
